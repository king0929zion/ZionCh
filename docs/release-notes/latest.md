## ZionChat v0.40.0

- `ZiCode` 新增“模型先规划、再执行”的执行层：在同步完 README、Issues、PR、工作流等 GitHub 上下文后，会先让 ZiCode 默认模型生成一份 JSON 执行计划，再决定是否建分支、写文件、删文件、建 Issue、建 PR、触发工作流或创建 Release。
- 这让 `ZiCode` 不再只是关键词触发器：即使用户提示词没有明确写出所有动作，ZiCode 也能先根据仓库上下文推断更合适的下一步，再落到具体 GitHub tool 上执行。
- 文件执行链也一起增强：同一轮里现在可以同时处理删除文件和写入文件，且文件写入优先使用模型规划阶段给出的目标路径和内容，再回退到提示词代码块或内容生成。
- `ZiCode` 的结果交付入口继续收口：如果本轮新建了 Issue、PR、Release，会优先把这些真正的 GitHub 结果作为打开入口返回，而不是退回仓库主页。
- 继续保持前一轮的 UI 收口：ZiCode 输入框更贴底、阴影更轻、Markdown 走灰白黑单色渲染，代码块操作文案也会随中英文切换。
